1//! # Once Crate
2//!
3//! The `once` crate provides thread-safe data structures that are designed for write-once semantics,
4//! making them particularly useful for concurrent programming scenarios where data is computed once
5//! and then read many times.
6//!
7//! ## Key Components
8//!
9//! - [`OnceVec`]: A thread-safe vector that allows pushing elements in a way that ensures they are
10//!   safely visible to other threads.
11//! - [`OnceBiVec`]: A bidirectional vector built on top of `OnceVec` that supports negative indices.
12//! - [`MultiIndexed`]: A multi-dimensional array that allows efficient storage and retrieval of values
13//!   using multi-dimensional coordinates.
14//! - [`Grove`] and [`TwoEndedGrove`]: Specialized collections that support the implementation of the
15//!   other data structures.
16//!
17//! ## Concurrency Support
18//!
19//! This crate is designed with concurrency in mind and uses atomic operations and locks to ensure
20//! thread safety. It can be used with the `concurrent` feature to enable parallel operations via
21//! the `rayon` crate.
22//!
23//! ## Testing with Loom
24//!
25//! The crate supports testing with the `loom` concurrency testing framework. Loom tests can be run with:
26//! ```bash
27//! RUSTFLAGS="--cfg loom" cargo test --release --features loom -- loom
28//! ```

46#[cfg(test)]
47mod test_utils {
48    pub fn num_threads() -> usize {
49        std::thread::available_parallelism()
50            .map(std::num::NonZero::get)
51            .unwrap_or(2)
52    }
53
54    pub fn values_per_thread() -> usize {
55        #[cfg(not(miri))]
56        return 1000;
57
58        // Miri is slow, so we reduce the number of values per thread to speed up the tests.
59        #[cfg(miri)]
